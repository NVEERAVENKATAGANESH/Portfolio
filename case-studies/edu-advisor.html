EduAdvisor — Academic Advisor Application

A WCAG-compliant academic planning and communication portal built with semantic HTML and ARIA, enabling students to plan courses, communicate with advisors, and track degree progress.

2023 Solo Developer ~3 weeks HTML · ARIA · WCAG 2.1
Demo Source

The Problem

Academic advising portals are notorious for poor accessibility — small text, poor contrast, missing focus indicators, and no keyboard navigation. This project set WCAG 2.1 AA compliance as a hard requirement, not an afterthought.

Process Pipeline

Key Features

1

Semantic Structure

Every section uses appropriate landmark elements (nav, main, aside, footer). Headings follow a strict h1→h2→h3 hierarchy with no skipped levels.

2

ARIA Attributes

Live regions (aria-live) for form feedback, aria-expanded for collapsibles, aria-label on icon buttons, aria-describedby linking inputs to error messages.

3

Keyboard Navigation

All interactive elements reachable and operable via keyboard alone. Focus trap implemented in modals. Skip-to-content link as first focusable element.

4

Colour Contrast

All text/background combinations verified against WCAG AA (4.5:1 normal, 3:1 large). Status colours supplemented with icons, not colour alone.

5

Course Planner

Drag-and-drop course scheduling with keyboard alternative (move via arrow keys + Enter).

Key Learnings

  • Semantic HTML alone eliminates ~40% of accessibility issues before writing a single ARIA attribute.
  • aria-live regions require careful scoping — overly broad live regions cause screen reader verbosity.
  • Keyboard focus management in modals (focus trap + restore on close) is essential, not optional.
