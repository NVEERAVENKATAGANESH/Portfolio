Lane Line Detection System

A real-time computer vision pipeline that identifies road lane markings from dashcam footage using OpenCV, Canny edge detection, and probabilistic Hough Transform.

2023 Solo Developer ~3 weeks Python · OpenCV · NumPy
Live Demo Source Code

The Problem

Autonomous and driver-assistance systems need to reliably detect lane boundaries in real-time under varying lighting, road wear, and weather conditions. A purely geometric solution (no deep learning) must be computationally cheap enough to run frame-by-frame while remaining robust to noise and perspective distortion.

  • Lane markings vary in color, width, and continuity (dashed vs. solid)
  • Shadows, reflections, and road texture introduce false edges
  • The Region of Interest (ROI) must exclude sky, vehicles, and roadside clutter
  • Detected line segments must be averaged and extrapolated into smooth full-length guides

Approach

1

Pre-processing

Convert to grayscale and apply a 5×5 Gaussian blur to suppress high-frequency noise before edge detection. This reduces spurious edges from road texture and minor lighting variation.

2

Canny Edge Detection

Apply Canny with tuned low/high thresholds (50/150). The double-threshold hysteresis keeps only strong edges and edges connected to them, discarding isolated weak edges from noise.

3

Region of Interest Masking

Define a trapezoidal polygon covering only the road ahead (lower 60% of frame, narrowing to a vanishing point). Bitwise AND with the edge image removes edges outside the driving corridor.

4

Probabilistic Hough Transform + Line Averaging

HoughLinesP detects line segments. Segments are classified by slope sign (positive = right lane, negative = left lane). Each group is averaged into a single representative line then extrapolated from the bottom of the frame to the horizon.

5

Overlay & Video Loop

Detected lane lines are drawn in semi-transparent green on the original frame using weighted_img (alpha blending). The pipeline runs on every frame of the input video in sequence.

The system successfully detects both dashed and solid lane lines across standard highway footage with a clean overlay. Edge cases (sharp curves, low contrast) are documented as future work for a curved-lane Hough variant.

Key Learnings

  • ROI tuning matters most — a poorly shaped mask lets in more false edges than any edge-detection threshold
  • Slope filtering (discarding near-horizontal segments) is essential to clean up horizontal road cracks
  • Gaussian blur kernel size directly trades noise suppression for edge sharpness; 5×5 is a good default
  • Classical CV is still highly effective for constrained, well-lit driving footage and runs faster than any neural approach
