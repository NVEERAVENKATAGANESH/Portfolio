BenefitsConnect — Public Benefits Advisor

A citizen-facing government benefits portal featuring a 5-step eligibility wizard, a 12-program catalog with smart filtering, case worker approval workflows, and a live analytics dashboard.

2024 Solo Developer ~5 weeks HTML · CSS · JavaScript
Demo Source

The Problem

Millions of eligible citizens miss out on government benefits because program information is scattered, confusing, and difficult to navigate. Existing government portals are often inaccessible, use bureaucratic language, and offer no guidance on which programs a person actually qualifies for.

  • Citizens don't know what benefits they qualify for without reading many pages of eligibility rules
  • Application processes are disconnected — different portals per program
  • Case workers have no central tool to track, review, and approve applications
  • Program administrators lack real-time visibility into application volume and trends

Approach

1

Citizen-first UX Design

Prioritized plain language, large touch targets, and clear progress indicators. Adopted WCAG 2.1 AA accessibility standards throughout: semantic HTML, ARIA labels, and keyboard-navigable forms.

2

5-Step Eligibility Wizard

Built a multi-step form that collects household size, income, age, employment status, and special circumstances. After submission, a matching algorithm scores each of 12 benefit programs and surfaces only those the user likely qualifies for.

3

Program Catalog with Smart Filtering

A searchable, filterable catalog of 12 programs (housing, food, healthcare, education, etc.) with details, eligibility summaries, and one-click application start. Filters by category, income bracket, and family status.

4

Case Worker Approval Workflow

A protected case worker view shows a queue of pending applications with applicant details. Workers can approve, deny (with reason), or flag for follow-up. Status updates are reflected in the citizen view.

5

Live Analytics Dashboard

An admin dashboard with charts showing application volume per program, approval rates, average processing time, and demographic trends across all 12 benefit programs. Built with Chart.js.

Key Learnings

  • Government UX demands radical simplicity — every word of jargon replaced with plain language measurably reduces user drop-off in multi-step forms
  • Mobile-first is essential for civic tech; a significant portion of low-income users access services exclusively via smartphone
  • Role separation (citizen vs. case worker vs. admin) should be enforced at the routing level, not just hidden via CSS
  • LocalStorage state is sufficient for demo-scale applications but would need to be replaced with a backend DB for real deployments with privacy requirements
