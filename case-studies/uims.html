UIMS: County Inventory Management System

An asset-lifecycle platform for municipal government built with Django REST Framework and React, enabling role-scoped access and real-time inventory tracking.

2024 Solo Developer ~6 weeks Django · React · PostgreSQL
Live Demo Source Code

The Problem

Municipal governments often rely on spreadsheets or aging legacy software to track physical assets — vehicles, equipment, IT hardware — across departments. This leads to duplicate procurement, unknown asset locations, expired maintenance cycles, and zero audit trails. The result: wasted budget and compliance risk.

  • No centralized view of what assets exist, where they are, or their condition
  • Different departments with conflicting data and no access controls
  • Manual check-in/check-out processes with no history
  • No alerts for maintenance due dates or warranties expiring

Approach

1

Domain Research & Data Modeling

Mapped the asset lifecycle (procurement → deployment → maintenance → disposal). Designed a normalized PostgreSQL schema with assets, departments, users, assignments, and audit logs.

2

Role-Based API with Django REST Framework

Built a RESTful API with JWT auth, three roles (Admin, Manager, Viewer), and row-level permissions. Each role gets a scoped view — admins see everything, managers see their department, viewers read-only.

3

React Dashboard

Built a component-based React frontend with a dashboard, asset CRUD, assignment workflow, and a filterable asset table. Used React Query for server state and Context API for auth.

4

Audit Logging & Notifications

Every create/update/delete action is logged to an immutable audit table with user, timestamp, and diff. Maintenance due dates trigger in-app alerts.

Tech Stack

Python / Django Django REST Framework React React Query JWT Auth PostgreSQL Vercel (Frontend) Role-Based Access Control

The system demonstrates production-ready patterns: JWT refresh tokens, RBAC, query optimization with select_related/prefetch_related, and a CI-deployable frontend on Vercel.

Key Learnings

  • RBAC at the ORM layer — filtering querysets per role prevents data leaks better than UI-only hiding
  • React Query dramatically simplifies server state management vs. raw useEffect + useState
  • Audit logs need to be append-only; regular FK deletes can cascade and destroy history
  • Gov tech UX requires clarity over cleverness — dense data tables beat fancy charts for daily operations
