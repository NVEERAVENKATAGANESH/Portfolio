Personal Portfolio — Architecture & Design

A fully hand-coded portfolio site built with vanilla HTML, CSS, and JavaScript — featuring GSAP scroll animations, a Three.js 3D hero sphere, PWA offline support, dark/light theming, and a CSS Grid timeline layout.

2025 Solo Developer Ongoing HTML · CSS · Vanilla JS · GSAP · Three.js
Demo Source

The Problem

Most portfolio templates are either too generic (template-based) or too complex (framework-heavy). The goal was to build a high-quality, performant portfolio from scratch — no React, no webpack — using only browser-native APIs and carefully chosen CDN libraries. Every design decision had to be defensible and lightweight.

Key Decisions

1

CSS Custom Properties System

All colours, spacing, and transitions defined as tokens in tokens.css. A single data-theme attribute on the body switches the entire palette without JavaScript style manipulation.

2

GSAP + ScrollTrigger

Entrance animations for every section use GSAP ScrollTrigger, respecting prefers-reduced-motion. Stagger delays create wave effects on grid items.

3

Three.js Hero Sphere

An icosahedron with vertex shader distortion and point cloud overlay creates the interactive 3D hero graphic, controlled by mouse position via raycasting.

4

PWA + Service Worker

A cache-first service worker enables full offline access. Manifest + theme-color metadata enables Add to Home Screen on mobile.

5

CSS Grid Timeline

The Journey section uses a two-column CSS Grid with a centred spine column, animating cards in from alternating sides via IntersectionObserver.

6

Performance

All scripts are defer-loaded in dependency order. Images use loading=lazy. No render-blocking CSS beyond a single minified bundle per concern.

Key Learnings

  • CSS Custom Properties make theming trivial but require discipline to keep token names semantic, not descriptive (--primary not --indigo-600).
  • defer script ordering is a footgun — Three.js must load before main.js or the sphere fails silently.
  • IntersectionObserver is more reliable than scroll event listeners for entrance animations, especially on mobile where scroll events fire irregularly.
