MediConnect — Online Doctor Consultancy

A patient-facing healthcare portal enabling primary medical consultation scheduling and emergency triage guidance, built entirely with vanilla HTML, CSS, and JavaScript.

2023 Solo Developer ~4 weeks HTML · CSS · JavaScript
Live Demo Source Code

Problem Statement

Patients in underserved areas struggle to access timely medical advice. Emergency triage decisions often require clinical help that isn't immediately available. A lightweight no-backend portal can bridge this gap by providing consultation scheduling and a symptom-based triage guide.

The goal was to demonstrate that a purely client-side application — with zero server infrastructure — could deliver a meaningful improvement in healthcare accessibility. By focusing on consultation scheduling and a structured triage decision aid, MediConnect shows that even resource-constrained deployments can provide real clinical value.

Key Features

1

Doctor Directory

Searchable list of specialists with profile cards, availability tags, and booking flow. Users can filter by speciality (GP, Cardiology, Dermatology, etc.) and see at-a-glance availability status before committing to a booking.

2

Appointment Booking

Form-based scheduling with a date/time picker and a confirmation modal. Submitted appointments are persisted in LocalStorage so the user's upcoming consultations survive page refresh without any backend infrastructure.

3

Emergency Triage Guide

Symptom checklist that categorises severity into green (self-care), yellow (book within 24 hrs), and red (seek emergency care immediately). Clear colour coding and plain-language instructions reduce cognitive load in stressful moments.

4

Responsive Design

Fully fluid layout optimised for mobile-first healthcare access. Touch targets exceed 44px, font sizes remain legible at low screen resolutions, and the triage guide works entirely offline once cached by the browser.

User Flow

The core patient journey is intentionally linear to minimise drop-off and decision fatigue:

Landing
Doctor Search
Profile View
Book Appointment
Confirmation

The triage guide sits on a separate, always-accessible path from the landing page — deliberately kept one click away so patients in distress are never more than two actions from critical guidance.

Tech Stack

Deliberately kept dependency-light to ensure the application is fast on low-bandwidth connections and deployable as static files on any hosting platform.

Key Learnings

UX clarity is critical in healthcare. Every label and error message must be unambiguous. Ambiguous form labels caused confusion in early testing — replacing "Select slot" with "Choose your preferred appointment time" eliminated most misfills.

LocalStorage is sufficient for prototype persistence. For a static proof-of-concept, LocalStorage provides a surprisingly capable appointment store. Its limitations (single-device, no cross-session sharing) become the next natural argument for a real backend.

Triage colour-coding dramatically improves comprehension speed. User observation showed that patients could identify their severity category within two seconds when colour was paired with an icon (green checkmark, yellow warning, red cross) — significantly faster than text-only layouts.
