3D Model Builder — Hierarchical OpenGL Modelling

An interactive 3D modelling application built in C++ with OpenGL and FLTK, supporting .ply mesh loading, hierarchical scene graphs, animated characters with joint transforms, and real-time transformation controls.

Problem

Teaching 3D graphics concepts requires hands-on tools that expose the rendering pipeline without hiding it behind a game engine. Building a modelling tool from first principles — with explicit matrix stacks, scene graphs, and animation loops — reinforces understanding of transforms, normals, and the OpenGL pipeline at every stage.

Key Features

1

.ply Mesh Loader

Custom parser reads ASCII/binary .ply files, extracts vertex positions, normals, and face indices, and uploads them to GPU via VBOs. Supports models up to 500k faces without performance degradation.

2

Hierarchical Scene Graph

Nodes can be parented to other nodes. Transformations compose correctly via matrix multiplication through the tree, enabling complex assemblies (e.g. robot arm with elbow and wrist joints).

3

Animated Characters

Keyframe animation system interpolates joint rotations using quaternion SLERP. Characters loop walk/idle animations at configurable playback speed.

4

Transformation Controls

FLTK sliders and buttons for translate, rotate, scale on selected nodes. Undo stack preserves the last 10 transform states.

5

Shading Modes

Toggle between flat, Gouraud, and Phong shading. Normal visualisation mode for debugging mesh loading artifacts.

Key Learnings

GLM's mat4 stack makes hierarchical transforms straightforward but column-major ordering is a constant source of bugs when mixing with row-major intuition.

Quaternion SLERP is essential for smooth joint animation — Euler interpolation causes gimbal lock at 90° joints.

.ply binary format is 4× faster to parse than ASCII for large meshes.
